En mi trabajo

Actualmente trabajo como Consultora en Accesibilidad Web en la Agencia Nacional de Discapacidad (Argentina) y en Concreta (Uruguay), y mis principales tareas son:

  • Revisión y evaluación de páginas web con sus respectivos informes técnicos basados en las WCAG.
  • Capacitaciones y asesoramiento sobre la temática a diferentes organismos que lo requieran.
  • Elaboración de material digital para las diferentes charlas y capacitaciones, esto puede incluir una presentación de diapositivas, documentos de texto, o programación de páginas web.
  • Mesas de trabajo o de consultas para diferentes organismos sobre sus proyectos en curso o en proceso.

Conceptos de utilidad

W3C (World Wide Web Consortium)
Es una comunidad internacional donde los miembros , el personal de tiempo completo y el público trabajan juntos para desarrollar estándares web.
WCAG (Web Content Accessibility Guidelines)
Los documentos WCAG explican cómo hacer el contenido web más accesible para las personas con discapacidad. Por “contenido” web se hace referencia, generalmente, a la información que se puede encontrar en una página o aplicación web, lo cual incluye textos, imágenes y sonidos, el código que define la estructura, la presentación, etc.
HTML (HyperText Markup Language)
Es el componente más básico de la Web. Define el significado y la estructura del contenido web. Además de HTML, generalmente se utilizan otras tecnologías para describir la apariencia/presentación de una página web (CSS) o la funcionalidad/comportamiento (JavaScript).
CSS (Cascading Style Sheets)
Es el lenguaje de estilos utilizado para describir la presentación de documentos HTML o XML. CSS describe como debe ser renderizado el elemento estructurado en la pantalla.
JavaScript (JS)
es un lenguaje de programación ligero, interpretado, o compilado justo-a-tiempo con funciones de primera clase. Si bien es más conocido como un lenguaje de scripting (secuencias de comandos) para páginas web, y es usado en muchos entornos fuera del navegador
